# 🛡️ StorCLI RAID Monitor

Веб-приложение для мониторинга состояния RAID-массивов на удалённом сервере через SSH + StorCLI.

## 📋 Возможности

- Подключение к серверу по SSH (ключ или пароль)
- Выполнение команды `storcli` и парсинг JSON-ответа
- Дашборд с информацией о:
  - Контроллере (модель, прошивка, статус)
  - Виртуальных дисках (VD) — RAID level, размер, состояние
  - Физических дисках (PD) — модель, размер, интерфейс
  - Топологии RAID-массива
  - Батарее (BBU)
- Цветовая индикация состояний (зелёный/жёлтый/красный)
- Кнопка ручного обновления данных
- Debug-режим с примером JSON (`Example.json`)

## 🚀 Быстрый запуск

### 1. Клонирование и переход в директорию

```bash
cd storecli
```

### 2. Создание виртуального окружения (опционально)

```bash
python -m venv venv

# Linux / macOS:
source venv/bin/activate

# Windows:
venv\Scripts\activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка конфигурации

```bash
# Скопируйте пример и заполните своими данными
cp .env.example .env

# Отредактируйте .env — укажите IP сервера, SSH-ключ/пароль и т.д.
```

### 5. Запуск приложения

```bash
python -m app.main
```

Приложение будет доступно по адресу: **http://localhost:8000**

## 🔧 Debug-режим

Для локальной разработки без реального сервера установите в `.env`:

```env
DEBUG_MODE=true
```

В этом случае бэкенд будет читать данные из файла `Example.json` вместо подключения по SSH.

## 🔌 API

### `GET /api/raid-status`

Возвращает JSON с обработанными данными RAID-массива.

**Успешный ответ (200):**
```json
{
    "success": true,
    "data": {
        "timestamp": "2026-04-05T15:41:00+00:00",
        "controller": { "model": "...", "status": "Optimal", "..." : "..." },
        "virtual_drives": [ { "dg_vd": "0/0", "type": "RAID10", "state": "Optl", "..." : "..." } ],
        "physical_drives": [ { "eid_slot": "35:0", "state": "Onln", "..." : "..." } ],
        "topology": [ { "dg": 0, "type": "RAID10", "state": "Optl", "..." : "..." } ],
        "bbu": [ { "model": "iBBU08", "state": "Optimal", "..." : "..." } ],
        "overall_status": "optimal"
    }
}
```

**Коды ошибок:**

| Код | Описание |
|-----|----------|
| 503 | Сервер недоступен по SSH |
| 502 | Ошибка выполнения команды storcli |
| 422 | Ошибка парсинга JSON-ответа |
| 500 | Непредвиденная внутренняя ошибка |

## 🔒 Безопасность

- Файл `.env` с учётными данными SSH **не включён** в Git (есть в `.gitignore`).
- SSH-подключение поддерживает аутентификацию по ключу (рекомендуется) и по паролю.
- Бэкенд не хранит учётные данные в коде.
